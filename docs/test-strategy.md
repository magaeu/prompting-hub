## Test Strategy

As Software QA Engineer I want to create a test strategy for the application under test. So we can share it with practioners.

To create the test strategy you will use heuristics from James Bach available on this [site](https://www.satisfice.com/download/heuristic-test-strategy-model)

Inputs for test strategy, according to the heuristic are: project enviorment, product factors and quality criteria. Follow each group and item present in the heuristic and provide a structure consice document. For structure the document use this [site](https://www.testmuai.com/blog/blueprint-for-test-strategy/) and this [site](https://medium.com/@rennanreis/qa-test-strategy-framework-how-i-built-a-real-world-manual-testing-portfolio-with-istqb-templates-39bff28f8926) as guide.

If any item is missing or any information is incomplete or needs clarification, mark the item and give an explation for it. Ask any important question during the process.

INPUT:

Test application with required information
