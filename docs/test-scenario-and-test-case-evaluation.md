### Test scenario and test case evalution

As Software QA Engineer I want to evaluate test cases for a feature so I can spot missing coverage and blind spots.

Review created test cases based on testing mnemonics SFDIPOT available on this [site](http://bit.ly/sfdipot_m). Test scenario and test case are created using BDD (Behavior Driven Development).

As output provide for each test scenario and/or test case an evaluation of test type (smoke, sanity, end-to-end), completeness, coverage, risk, and adherence to BDD.

If anything is missing mark it and give a proper explanation.

INPUT
[feature]
[created test cases]
