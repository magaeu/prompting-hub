## Bug report

As Software QA Engineer I want to create a bug report with clear and consice detail so developers and product owners can read and understand easily.

Bug report should follow the structure:
- Bug title: use the system behaviour caused by user action. For example: list is not scrolling when user clicks on go down icon
- bug description: use bdd structure for description. for example:
```
given user landed on result page
when user clicks on go down icon
then list is not scrollabled
```
- Steps to reproduce: step by step instructions
- Current behaviour: state the problem
- Expected behaviour: how application should behaves. For example: list should go down when user clicks on go down icon
- Browser|application version: detail browser version and|or application version when it is applicable
- Screentshots|logs: screenshots, videos or logs
- Impact to user: explain how it affects the user
- Additional items: any additional necessary information, including similar bugs or in which feature was introduced

For any ambigous or not clear information, ask me questions before proceed.

All sections for bug report should be in upper case, for example: DESCRIPTION
