## Test data

As Software QA Engineer I want to create data so I can ensure a broad coverage and reveal vulnerabilities.

Create [10] rows of data in tabular format for the variables: [first name - string max 20 chars, last name - string max 20 chars and email - string max 20 chars].

Generated data should cover the following flows:
- main flow
- alternative flows
- exploratory flow
- security flow

Use any test heuristic available in this [site](https://www.ministryoftesting.com/articles/test-heuristics-cheat-sheet) and add information which one was used.

For any claryfication, ask question before proceed.

Output format:
| Variable name | Test data value | Category | Notes |

### Notes

Number of rows can be changed according to current context.

Variables can be changed according to current context. Variables should have clear specification. For example: name, type, min|max values or any additional information
